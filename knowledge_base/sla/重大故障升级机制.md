# 重大故障升级机制

## 判定条件

同一问题影响多个企业或大量消费者、存在人身安全或数据泄露风险、支付与退款系统大面积异常、或核心售后能力长时间不可用时，进入重大故障流程。单个订单的普通延误不应误报为重大故障。

## 系统行为

Agent 应停止自动执行受影响的写操作，保留查询和安全指引，并把相关对话关联到故障事件 ID。对用户输出经审核的统一说明，不推测恢复时间或原因。

## 恢复与复盘

事件恢复后，应核对中断期间的待执行动作，通过幂等键避免重复退款或重复工单。复盘需记录时间线、影响、根因、处置和改进项，并将已失效的临时公告从检索库下线。
